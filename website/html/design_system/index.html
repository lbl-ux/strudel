What is a Design System

Scientists frequently encounter intricate information that necessitates the use of complex interfaces. Too often the way these interfaces perform similar tasks or present similar information is not consistent between different applications, or sometimes even within a single application.
A scientific Design System will provide projects with comprehensive guidelines & exemplify patterns which are relevant across multiple scientific domains.

Design Systems offer rules and guidelines for designing and implementing software user interfaces for a project.

  • Visual Interface fundamentals like grid, theme, colors, typography and layout
  • Visual components like buttons, forms etc. which are building blocks of an interface
  • Common patterns like login, search, notification, filtering, etc
  • Interaction patterns like taking actions, getting more information, navigating through results, and navigating through the interface

Using these guidelines, science teams will be able to design and implement more usable & effective interfaces that use consistent styles and behaviors to address their complex workflow requirements.

Task Flows

Our design system will be framed and guided by Task Flows, which are a set of steps (represented by a series of screens) that help to accomplish a task and represent how a user progresses through a UI.

The task-flows come with corresponding React templates found within our STRUDEL UI library. This library is constructed using the React JavaScript framework and is built on the foundation of the widely used Material UI (MUI) Components library.

You can access these coded templates on our STRUDEL UI Github repository: https://github.com/lbl-ux/strudel-ui.
