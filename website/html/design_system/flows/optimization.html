TASK FLOW

Running Optimizations / Simulations / Analyses

NOTE: This is a work in progress and is expected to frequently change.

Intent of task flow:

Enable users to run calculations/ optimizations/ simulations on a pre-defined/ newly created/ altered model with attributes to observe and compare results to help users make data-driven decisions/ improve performance/ reduce costs/ increase safety/ reduce risk in a variety of fields and applications

Task flow example template

View in Figma

Common UI patterns used:

Lists, stepper/progress indicator, forms, data tables, panels

Optimizations usually require a stepper also known as progress indicator pattern to guide users through a long multi-step process. Complex optimizations require user input for multiple attributes and involve different types of data organization, such as forms, tables, lists, and panels. These elements can be grouped into user-friendly sections using tabs.

Other UI patterns used:

Interactive canvas, diagrams, tabs, charts & graphs, search, etc.

They might have complex inputs which may need uncommon patterns like canvas interactions for manipulating attributes in diagrams. Optimization results may have visualizations, such as explorative maps, charts, graphs, and diagrams, to provide better understanding of the data, making it easier for users to identify key insights and draw meaningful conclusions.

Do's:

  • Break the task flow into multiple workable steps and use a progress indicator/ stepper to help users see the progress and other steps in process to complete
  • Organizing the information into sections that are easy to digest, this helps improves the readability and searchability
  • Offer guidance, tips and links to detailed documentation for complex inputs & interactions which may not be intuitive to many users
  • If it involves long forms for input - pre-fill the forms with default values wherever possible
  • Make attributes searchable and filterable to make users easy to get to attributes of their interests
  • Give real time error warnings for mis-configurations - especially for long complex forms.
  • Users should be able to recover from failed simulations, by letting them go back and change configurations

Dont's:

  • Don’t break the workflow into too many small steps making it a long process to complete, it may overwhelm the users
  • Don’t assume users would understand each input field and the values to configure it
