TASK FLOW

Comparing Scenarios

Intent of task flow:

To help decision making through comparison of complex entities with multiple attributes, the comparison workflows lay out the information in a clear and concise manner which can save users time and effort in researching and analyzing different options.

Task flow example template

Common UI patterns used:

Lists, data tables, search, filter
The most common way to show comparison between different options with multiple attributes is through a data table, with the rows and columns as attributes and options.

Other UI patterns used:

Visualizations, charts & graphs, diagrams etc.
In addition to the tabular format, visual comparisons through charts/ graphs and dimensions for main attributes help to highlight the key differences which makes it faster and easier

Do’s:

  • Understand how complex/ easy the scenario manipulation for comparison is. Based on that - the comparison process can involve a step for selecting attributes creating new scenarios to compare; or can mean selecting previously created scenarios to be compared against each other
  • Prioritize the attributes of scenarios which are critical for comparison instead of exposing all of the attributes at the same level.
  • If there are too many attributes for comparison, give a way for users to search & select the attributes from the full list which they want to compare at priority. This makes it easy for them to focus on their interests instead of getting overwhelmed and scan the large comparison matrix
  • Consider what way of showing diagrams/maps for scenario comparison would be effective - side by side, one at a time, sliders, overlay etc.
  • Using data visualizations with comparison metrics will help to highlight the differences and similarities of scenarios
  • Give a way to save/ export comparison results for user’s future reference
