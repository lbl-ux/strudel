Task Flows

A user's flow through a Science UI when trying to accomplish particular scientific tasks are often distinct compared to commercial software and often emerge across scientific domains.
We focused our initial efforts developing these flows into our design system, which we refer to as Task Flows. Multiple task flows can be combined to create a formal workflow that can be represented as a DAG computationally.

We have documented several recurring task flows found in scientific applications. Our characterizations come from our team's extensive experience and observations in the field, supplemented by heuristic evaluations of prioritized scientific software.

The documentation for each task flow includes a standard visual template and a set of user experience design guidelines

  • Optimization. Running Optimizations / Simulations / Analyses
  • Comparison. Comparing Scenarios/ Analyses / Simulations
  • Exploring Entities. Exploring Entities/ Data List / Collections
  • Dataset. Finding, Exploring, and Downloading Datasets
  • Contributing Data. Contributing Data/ Dataset / Collections
  • Monitoring. Monitoring Tasks / Experimental Events / Data
  • Analyzing. Analyzing Data (Coming Soon)
  • History. Tracking History/ Lab Notebook (Coming Soon)
  • Flowsheets. Creating Flowsheets (Coming Soon)
  • Accounts. Managing Accounts (Coming Soon)
