Exploring Entities

NOTE: This is a work in progress and is expected to frequently change.

Intent of task flow:

Many scientific softwares exist to make domain specific data available and accessible to the community for enabling more research in that domain

Task flow example template

View in Figma

Common UI patterns used:

Data tables,summary view, details view, search, filters
The most common way to display an organized entity repository is via data tables. By incorporating search, sort, and filter functions, data tables enhance data accessibility and facilitate efficient data analysis

Other UI patterns used:

Interactive Maps, charts/ graphs, diagrams, etc.
Visualizations, such as explorative maps, charts, graphs, and diagrams, are effective tools that can be used in conjunction with data tables to provide users with a more comprehensive understanding of the data in their respective contexts. These visualizations can help to highlight relevant patterns and narrow down the scope of the data, making it easier for users to identify key insights and draw meaningful conclusion

Do’s:

  • Search and filter capabilities is really important to give a good exploration experience
  • Data table: Give pagination and customisable number of rows to view in one page
  • Link the entity name/ identifier to entity summary/ detail view
  • Entity detail view: Give proper title and breadcrumb navigation to go back to main data table
  • Entity detail view: Consider giving page navigation if too many sections
