Background

What is a Typology

A sociotechnical typology of scientific work enables researchers to systematically analyze & compare diverse scientific projects. This analysis helps surface commonalities that impact software being produced and used by different domains and organizations.

The STRUDEL Typology is a tool for categorizing key elements making up scientific projects building and using software products. It is designed to help users identify and compare common activities in scientific work that should be considered when focusing on user experiences, software sustainability, and more.

Who can use the Typology

The STRUDEL typology is intended to support a range of users with their work. Key examples include:

  • Funding Agencies — Staff can use the typology to evaluate proposed projects and request investment or planning for UX, software sustainability, and so on.
  • Project Leadership — Project leaders can identify opportunities for UX, Software Sustainability, Community Engagement, etc. and determine when and how much to invest over a project's life cycle.
  • Project Domain Experts — Scientists on the project can identify & give feedback on common workflows to help specify requirements for the software products being developed.
  • Project Developers — Project developers can use the typology to identify common task flows & examine which pieces of the Design System to start their work with.

How we developed the Typology

The STRUDEL team developed this typology through reviewing years of its user experience research products, the sociotechnical literature, examination of diverse project planning documents and products, and interviews with project leaders.

STRUDEL Typology Overview

To help teams conduct effective UX work a typology of scientific software work needs to help explore relationships between three key sociotechnical elements:

  • Organizational Issues — the bounding, bedrock contexts shaping the work at hand
  • Social Issues — the dynamics of people arranged into teams doing work
  • Technical Issues — the details shaping software product(s) & affecting a user’s experience

Drawing upon these three overarching elements we have defined three top level facets to organize the various dimensions and questions. These include Project Composition arranging concerns around organizational issues, People & Teams capturing the social concerns, and Software Product(s) to structure technical issues.
